## Terraform 

- It is an **Infrastructure as Code (IaC)** tool developed by **HashiCorp**.
- It allows you to create, update, and manage infrastructure using configuration files instead of manually creating resources.

---

## Infrastructure as Code (IaC)

- Managing infrastructure using code instead of manual processes.
- Instead of clicking buttons in a cloud console, you write code to create and manage resources.

Example resources: Virtual Machines (EC2), Networks (VPC), Storage (S3), Databases (RDS), Load Balancers

---

## Why Use Terraform?

Without Terraform:

* Create resources manually.
* Repeat the same steps every time.
* Difficult to maintain.
* Higher chance of human errors.

With Terraform:

* Infrastructure is created automatically.
* Configuration is reusable.
* Easy to update and maintain.
* Faster deployments.
* Consistent infrastructure.

## How Terraform Works

```text
Write Configuration
        │
        ▼
terraform plan
        │
        ▼
Review Changes
        │
        ▼
terraform apply
        │
        ▼
Infrastructure Created
```

---

## Terraform Configuration Example

```hcl
provider "aws" {
  region = "us-east-1"
}

resource "aws_instance" "web" {
  ami           = "ami-xxxxxxxx"
  instance_type = "t2.micro"
}
```

This configuration creates an AWS EC2 instance.
