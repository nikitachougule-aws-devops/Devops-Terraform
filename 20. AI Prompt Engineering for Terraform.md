## AI Prompt Engineering for Terraform

- **Prompt engineering** is the process of writing clear and specific instructions for AI to generate accurate Terraform code, explanations, or solutions.
- A good prompt produces better results.

---

## Why Use Prompt Engineering?

Good prompts help AI:

* Generate correct Terraform code.
* Explain Terraform concepts.
* Fix errors.
* Create reusable modules.
* Improve existing configurations.
* Save development time.

---

## Structure of a Good Prompt

A good prompt includes:

* **Task** – What you want AI to do.
* **Technology** – Terraform and cloud provider.
* **Requirements** – Specific details.
* **Output** – Expected result.

Example:

```text
Create Terraform code to launch an AWS EC2 instance.

Requirements:
- Region: us-east-1
- Instance type: t2.micro
- Use variables
- Add tags
- Include outputs
```

## Best Practices

* Be specific about the cloud provider (AWS, Azure, or GCP).
* Mention the required resources.
* Ask for variables, outputs, and modules when needed.
* Paste the complete error message for troubleshooting.
* Ask AI to follow Terraform best practices.
* Always verify the generated code using:

  * `terraform fmt`
  * `terraform validate`
  * `terraform plan`

## Key Points

* Prompt engineering is the practice of writing effective instructions for AI.
* Clear and detailed prompts produce better Terraform code.
* AI can generate code, explain concepts, review configurations, and troubleshoot errors.
* Always validate AI-generated code before applying it.
* Use AI to improve productivity while understanding the Terraform code it generates.
