## ✤ Basic Commands

### Initialize a Project

```bash
terraform init
```

Initializes the Terraform project by downloading the required providers and creating the `.terraform` directory.

---

### Validate Configuration

```bash
terraform validate
```

Checks the Terraform configuration for syntax errors.

---

### Format Configuration

```bash
terraform fmt
```

Formats Terraform configuration files according to the standard style.

---

### Preview Changes

```bash
terraform plan
```

Shows what Terraform will create, update, or delete without making any changes.

---

### Apply Changes

```bash
terraform apply
```

Creates or updates infrastructure based on the configuration.

Apply without confirmation:

```bash
terraform apply -auto-approve
```

---

### Destroy Infrastructure

```bash
terraform destroy
```

Deletes all resources managed by Terraform.

Destroy without confirmation:

```bash
terraform destroy -auto-approve
```

---

## ✤ State Commands

### Show Current State

```bash
terraform show
```

Displays the current infrastructure managed by Terraform.

---

### List Resources

```bash
terraform state list
```

Lists all resources stored in the state file.

---

### Show a Resource

```bash
terraform state show <resource-name>
```

Example:

```bash
terraform state show aws_instance.web
```

Displays detailed information about a specific resource.

---

## ✤ Output Commands

### View Outputs

```bash
terraform output
```

Displays all output values.

View a specific output:

```bash
terraform output <output-name>
```

---

## ✤ Workspace Commands

### List Workspaces

```bash
terraform workspace list
```

---

### Create a Workspace

```bash
terraform workspace new dev
```

---

### Switch Workspace

```bash

```

---

### Show Current Workspace

```bash
terraform workspace show
```

---

## ✤ Provider Commands

### Display Providers

```bash
terraform providers
```

Lists the providers used in the project.

---

## Refresh State

```bash
terraform refresh
```

Updates the state file to match the actual infrastructure.

> **Note:** In newer Terraform versions, `terraform refresh` is deprecated. Running `terraform apply` or `terraform plan` refreshes the state automatically.

---

## ✤ Version Information

```bash
terraform version
```

Displays the installed Terraform version.

---

## ✤ Help

```bash
terraform help
```

Displays all available Terraform commands.

Help for a specific command:

```bash
terraform plan -help
```
