## AI + Terraform Project

## Project Overview

In this project, you will use **AI** to generate, review, and improve Terraform code while deploying infrastructure on AWS.

This project demonstrates how AI can speed up Terraform development and improve code quality.

---

## Project Goal

Deploy a simple AWS infrastructure containing:

* EC2 Instance
* Security Group
* Variables
* Outputs

Use AI to:

* Generate Terraform code.
* Review the code.
* Fix errors.
* Suggest best practices.

### Step 1: Generate Code with AI

Prompt:

```text
Generate Terraform code to create an AWS EC2 instance.

Requirements:
- Region: us-east-1
- Instance type from a variable
- Security Group allowing SSH
- Tags
- Outputs for Instance ID and Public IP
```

Review the generated code before using it.

---

### Step 2: Initialize the Project

```bash
terraform init
```

Downloads the required providers.

---

### Step 3: Format the Code

```bash
terraform fmt
```

Formats all Terraform files.

---

### Step 4: Validate the Configuration

```bash
terraform validate
```

Checks for syntax and configuration errors.

---

### Step 5: Preview Changes

```bash
terraform plan
```

Shows the resources Terraform will create.

---

### Step 6: Deploy Infrastructure

```bash
terraform apply
```

Creates the AWS resources.

---

### Step 7: Review with AI

Prompt:

```text
Review my Terraform code for:

- Best practices
- Security issues
- Hardcoded values
- Module opportunities
- Performance improvements

Explain every suggestion.

<paste code>
```

AI can recommend improvements before deployment.

---

### Step 8: Destroy Resources

```bash
terraform destroy
```

Deletes all infrastructure created by Terraform.

## Best Practices

* Review AI-generated code before using it.
* Use variables instead of hardcoded values.
* Keep secrets out of Terraform files.
* Run `terraform fmt` and `terraform validate`.
* Always review `terraform plan` before `terraform apply`.
* Delete unused resources with `terraform destroy` to avoid cloud costs.

---

## Key Points

* AI helps generate, review, and improve Terraform code.
* Terraform provisions and manages cloud infrastructure.
* Always validate AI-generated code before deployment.
* Follow the workflow: **Generate → Review → Init → Fmt → Validate → Plan → Apply**.
* AI improves productivity, but final verification is the developer's responsibility.
