## Terraform Troubleshooting

### 1. `terraform init` Fails

**Problem:** Terraform cannot initialize the project.

**Possible Causes:**

* No internet connection.
* Incorrect provider configuration.
* Invalid backend configuration.

**Solution:**

```bash
terraform init
```

Check the error message and verify your provider or backend configuration.

---

### 2. Provider Plugin Not Found

**Problem:** Terraform cannot download the required provider.

**Solution:**

Run:

```bash
terraform init
```

If needed, delete the `.terraform` folder and initialize again.

```bash
rm -rf .terraform
terraform init
```

---

### 3. Invalid Configuration

**Problem:** Terraform reports syntax or configuration errors.

**Solution:**

```bash
terraform validate
```

Fix the reported errors and validate again.

---

### 4. Wrong Formatting

**Problem:** Configuration files are not properly formatted.

**Solution:**

```bash
terraform fmt
```

---

### 5. Authentication Failed

**Problem:** Terraform cannot connect to the cloud provider.

**Possible Causes:**

* Invalid credentials.
* Expired access keys.
* Incorrect profile.

**Solution:**

* Verify your cloud credentials.
* Check the configured profile.
* Ensure the user has the required permissions.

---

### 6. Resource Already Exists

**Problem:** Terraform tries to create a resource that already exists.

**Solution:**

Import the existing resource into Terraform.

```bash
terraform import <resource_type>.<resource_name> <resource_id>
```

Example:

```bash
terraform import aws_instance.web i-0123456789abcdef0
```

---

### 7. State File Locked

**Problem:** Another Terraform operation is using the state file.

**Solution:**

Wait for the current operation to finish.

If the lock is stuck:

```bash
terraform force-unlock <LOCK_ID>
```

> Use `force-unlock` only if you are sure no other Terraform operation is running.

---

### 8. Resource Drift

**Problem:** Someone changed the infrastructure manually outside Terraform.

**Solution:**

Run:

```bash
terraform plan
```

Review the differences and apply the required changes.

```bash
terraform apply
```

---

### 9. Wrong Workspace

**Problem:** Resources are created in the wrong environment.

**Solution:**

Check the current workspace:

```bash
terraform workspace show
```

Switch if required:

```bash
terraform workspace select dev
```

---

### 10. Destroy Failed

**Problem:** Terraform cannot delete a resource.

**Possible Causes:**

* Resource dependency.
* `prevent_destroy` is enabled.
* Insufficient permissions.

**Solution:**

* Check the error message.
* Remove dependencies if required.
* Disable `prevent_destroy` if appropriate.
* Verify cloud permissions.

---

### 11. Variables Not Loaded

**Problem:** Terraform asks for variable values.

**Solution:**

Use a variable file:

```bash
terraform apply -var-file="terraform.tfvars"
```

Or pass variables directly:

```bash
terraform apply -var="instance_type=t3.micro"
```

---

### 12. Backend Configuration Changed

**Problem:** Backend configuration has been modified.

**Solution:**

Reinitialize Terraform.

```bash
terraform init -reconfigure
```

---

### 13. Accidentally Deleted the State File

**Problem:** `terraform.tfstate` is missing.

**Solution:**

* Restore it from backup.
* If using a remote backend, reinitialize:

```bash
terraform init
```

---

### 14. Wrong Terraform Version

**Problem:** Project requires a different Terraform version.

**Solution:**

Check the version:

```bash
terraform version
```

Install the required version and run:

```bash
terraform init
```

---

### 15. Apply Fails

**Problem:** `terraform apply` stops with an error.

**Solution:**

* Read the error message.
* Fix the configuration or permissions.
* Run:

```bash
terraform plan
terraform apply
```

again.

---

| Problem                 | Solution                                                    |
| ----------------------- | ----------------------------------------------------------- |
| `terraform init` fails  | Check provider/backend and run `terraform init`             |
| Provider not found      | Delete `.terraform` and run `terraform init`                |
| Invalid configuration   | `terraform validate`                                        |
| Wrong formatting        | `terraform fmt`                                             |
| Authentication failed   | Verify cloud credentials                                    |
| Resource already exists | `terraform import`                                          |
| State file locked       | `terraform force-unlock`                                    |
| Resource drift          | `terraform plan` → `terraform apply`                        |

| Destroy fails           | Check dependencies and permissions                          |
| Variables not loaded    | Use `-var` or `-var-file`                                   |
| Backend changed         | `terraform init -reconfigure`                               |

| Apply fails             | Fix errors and rerun `terraform plan` and `terraform apply` |

---

## Key Points

* Use `terraform validate` to find configuration errors.
* Use `terraform fmt` to format configuration files.
* Use `terraform plan` before `terraform apply`.
* Use `terraform import` to manage existing resources.
* Use `terraform force-unlock` only when a state lock is stuck.
* Always verify the active workspace before applying changes.
* Keep state files backed up or use a remote backend.
