## AI Code Review for Terraform

**AI Code Review** is the process of using AI to review Terraform code and identify:

* Syntax errors
* Best practice improvements
* Security issues
* Performance improvements
* Code duplication
* Readability issues

AI helps improve code quality before deployment.

---

## Why Use AI for Code Review?

AI can help you:

* Detect mistakes quickly.
* Suggest best practices.
* Improve readability.
* Identify security risks.
* Recommend reusable modules.
* Reduce manual review effort.

---

## What AI Can Review

AI can review:

* Providers
* Resources
* Variables
* Outputs
* Modules
* State configuration
* Backend configuration
* Lifecycle rules
* Provisioners

## Common Review Areas

### 1. Code Formatting

Check if the code follows Terraform formatting standards.

Example command:

```bash id="y9qq4o"
terraform fmt
```

---

### 2. Configuration Validation

Check for syntax errors.

```bash id="kqv38v"
terraform validate
```

---

### 3. Hardcoded Values

Bad Example:

```hcl id="mmp9w8"
instance_type = "t2.micro"
```

Better:

```hcl id="pjlwm8"
instance_type = var.instance_type
```

Use variables instead of hardcoded values.

---

### 4. Resource Naming

Bad:

```hcl id="55pxrn"
resource "aws_instance" "a" {}
```

Better:

```hcl id="4g5u9h"
resource "aws_instance" "web_server" {}
```

Use meaningful resource names.

---

### 5. Security Review

AI checks for:

* Hardcoded passwords
* API keys
* Secret values
* Open security groups
* Publicly exposed resources

Bad Example:

```hcl id="c6mhbc"
password = "Admin123"
```

Better:

```hcl id="d5i2hc"
password = var.db_password
```

---

### 6. Module Suggestions

If similar code is repeated, AI may recommend creating a module.

Example:

```text id="m8xkkd"
Repeated EC2 configuration
        │
        ▼
Create an EC2 Module
```

---

### 7. Output Review

AI checks whether important outputs are defined.

Example:

```hcl id="d4j9y0"
output "public_ip" {
  value = aws_instance.web.public_ip
}
```

---

### 8. Provider Version

Instead of leaving provider versions unrestricted:

```hcl id="hcckgq"
terraform {
  required_providers {
    aws = {
      source = "hashicorp/aws"
    }
  }
}
```

Better:

```hcl id="m7chdq"
terraform {
  required_providers {
    aws = {
      source  = "hashicorp/aws"
      version = "~> 5.0"
    }
  }
}
```

## Best Practices

* Use variables instead of hardcoded values.
* Pin provider versions.
* Keep resources well organized.
* Use modules for reusable code.
* Avoid storing secrets in Terraform files.
* Run `terraform fmt` and `terraform validate`.
* Review the execution plan before applying changes.

---

## AI Limitations

* AI can miss cloud-specific requirements.
* AI suggestions may not always follow your organization's standards.
* Always verify generated or modified code before deployment.
* Test changes in a development environment first.

---

## Key Points

* AI can review Terraform code for quality, security, and best practices.
* It helps detect syntax errors, hardcoded values, and code duplication.
* AI can recommend variables, modules, outputs, and provider version constraints.
* Always validate AI-reviewed code using `terraform fmt`, `terraform validate`, and `terraform plan`.
* Review AI suggestions before running `terraform apply`.
