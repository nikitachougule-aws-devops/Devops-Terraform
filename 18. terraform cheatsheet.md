## Terraform Cheat Sheet

### Project Setup

```bash
terraform init              # Initialize project
terraform version           # Show Terraform version
terraform help              # Show help
```

---

### Configuration

```bash
terraform fmt               # Format files
terraform validate          # Validate configuration
```

---

### Planning & Deployment

```bash
terraform plan              # Preview changes
terraform apply             # Create/update infrastructure
terraform apply -auto-approve
terraform destroy           # Delete infrastructure
terraform destroy -auto-approve
```

---

### State Commands

```bash
terraform show                      # Show current state
terraform state list                # List resources
terraform state show <resource>     # Show resource details
terraform state rm <resource>       # Remove resource from state
terraform import <resource> <id>    # Import existing resource
```

---

### Variables & Outputs

```bash
terraform output                    # Show outputs
terraform output <name>             # Show specific output
terraform apply -var="key=value"
terraform apply -var-file="terraform.tfvars"
```

---

### Workspace Commands

```bash
terraform workspace list
terraform workspace show
terraform workspace new dev
terraform workspace select dev
terraform workspace delete dev
```

---

### Backend

```bash
terraform init -reconfigure
```

Reinitialize after backend configuration changes.

### Important Concepts

| Concept     | Purpose                                |
| ----------- | -------------------------------------- |
| Provider    | Connects Terraform to a cloud platform |
| Resource    | Infrastructure object                  |
| Variable    | Input value                            |
| Output      | Display resource information           |
| Module      | Reusable Terraform code                |
| State File  | Tracks infrastructure                  |
| Backend     | Stores the state file                  |
| Workspace   | Separate environment                   |
| Provisioner | Runs scripts after resource creation   |
| Lifecycle   | Controls resource behavior             |

* `terraform init` → Initialize the project.

* `terraform validate` → Check configuration syntax.
* `terraform plan` → Preview infrastructure changes.
* `terraform apply` → Create or update infrastructure.
* `terraform destroy` → Delete infrastructure.
* `terraform show` → Display the current state.
* `terraform output` → Display output values.
* `terraform import` → Import existing resources.
* `terraform workspace` → Manage multiple environments.
* `terraform.tfstate` → Stores infrastructure state.
