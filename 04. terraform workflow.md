## Terraform Workflow

- It is the step-by-step process of creating, updating, and managing infrastructure using Terraform.
- The typical workflow is: **Write → Initialize → Plan → Apply → Verify → Update → Destroy**

---

### Step 1: Write Configuration

- Create a Terraform configuration file (`main.tf`) and define the infrastructure.
- Example:

```hcl
provider "aws" {
  region = "us-east-1"
}

resource "aws_instance" "web" {
  ami           = "ami-xxxxxxxx"
  instance_type = "t2.micro"
}
```

---

### Step 2: Initialize Terraform

- Initialize the working directory.

```bash
terraform init
```

**What happens?**

* Downloads required providers.
* Creates the `.terraform` directory.
* Prepares the project for use.

---

### Step 3: Validate Configuration (Optional)

- Check for syntax errors.

```bash
terraform validate
```

- This ensures the configuration is valid before creating resources.

---

### Step 4: Preview Changes

- Generate an execution plan.

```bash
terraform plan
```

**What happens?**

* Compares your configuration with the current infrastructure.
* Shows what will be created, updated, or deleted.
* Makes no changes to the infrastructure.

---

### Step 5: Apply Changes

- Create or update infrastructure.

```bash
terraform apply
```
- Terraform displays the execution plan and asks for confirmation.
- Type:

```text
yes
```
- Terraform then creates or updates the resources.

---

### Step 6: Verify Resources

- Verify that the resources were created.
- Examples:

* Check the AWS Console.
* Run:

```bash
terraform show
```

---

### Step 7: Update Infrastructure

- Modify the configuration file.
- Example:

```hcl
instance_type = "t3.micro"
```
- Apply the changes:

```bash
terraform apply
```
- Terraform updates only the modified resources.

---

### 8: Destroy Infrastructure

- Delete all resources managed by Terraform.

```bash
terraform destroy
```
- Terraform asks for confirmation before deleting resources.

## Real-World Example

Suppose you want to create an AWS EC2 instance.

1. Write the EC2 configuration in `main.tf`.
2. Run `terraform init`.
3. Validate the configuration.
4. Run `terraform plan` to preview changes.
5. Run `terraform apply` to create the EC2 instance.
6. Modify the configuration if needed.
7. Run `terraform apply` again to update the infrastructure.
8. When no longer needed, run `terraform destroy` to remove all resources.

## Common Workflow Commands

| Command              | Purpose                         |
| -------------------- | ------------------------------- |
| `terraform init`     | Initialize the project          |
| `terraform validate` | Validate configuration          |
| `terraform plan`     | Preview changes                 |
| `terraform apply`    | Create or update infrastructure |
| `terraform show`     | Display current state           |
| `terraform destroy`  | Delete infrastructure           |

---

## Key Points

* Terraform workflow follows **Write → Init → Validate → Plan → Apply → Verify → Update → Destroy**.
* Always run `terraform init` before using a new project.
* Use `terraform validate` to check configuration syntax.
* Use `terraform plan` to preview changes before applying them.
* Use `terraform apply` to create or update infrastructure.
* Use `terraform destroy` to remove all managed resources.
